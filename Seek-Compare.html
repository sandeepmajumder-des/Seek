Whatfix Seek Design exploration

Two directions for how Seek lives on your desktop.

We've built both, end to end. Try each one, then tell us which mental model fits how you'd actually run tasks day-to-day. There are no wrong answers — the goal is to pick the one that feels native, not the one that demos best.

Option A Workspace

Full-screen workspace

Seek as its own app. Persistent sidebar, dedicated surfaces for every job — composer, activity, prompt library, decision trails. Built for users who want a control center.

  • Familiar workspace pattern. Sidebar + canvas, like Linear or Notion. Fast to onboard.
  • Dedicated surfaces. Tasks, Activity, Prompt Library — each gets the real estate it deserves.
  • Dense data views. Sortable activity table, decision-trail flowcharts, multi-column overview cards.
  • Takes the screen. Seek is primary; the SaaS app you're automating sits in a separate tab.
Try the workspace ~5 min walkthrough
Option B Companion

Floating companion panel

A 400px panel that floats over the app you're working in. Expands to 940px with the full task drawer. Designed to disappear when you don't need it.

  • 400 → 940 px. Mobile-sized by default; expands when you want the task list visible.
  • Floats over real work. Salesforce, Zendesk, Dynamics stay in view. Seek doesn't take over.
  • Execution-first. Sticky thinking bar, session recording, milestones. Minimal chat back-and-forth.
  • One surface, multiple views. Active task, recent runs, decision trails — all without leaving the panel.
Try the panel ~5 min walkthrough
Workspace fits when
You want Seek as a control center
  • You monitor many tasks at once and want them all visible.
  • You spend more time in Seek itself than in the apps it automates.
  • You want sortable tables, dense activity views, prompt library at scale.
  • The mental model is "open Seek to start a task" — like opening Linear.
Companion fits when
Seek runs alongside your real work
  • You live in Salesforce / Zendesk / Dynamics and want Seek there too.
  • One prompt at a time — execution matters, chat doesn't.
  • You want minimal context switch; Seek shouldn't replace your workspace.
  • The mental model is "Seek is here while I work" — like Cmd+K, always-on.

Both prototypes are fully functional HTML. Click through real flows: attach a flow, send a prompt, follow up on a stale task, open Activity, browse the Prompt Library. Feedback welcome on the Seek design channel.
